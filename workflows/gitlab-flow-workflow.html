GitLab Flow

A hybrid workflow combining feature branching with environment-based deployment. GitLab Flow enables controlled promotion of changes through dev, staging, and production branches.

Visual Workflow Diagram

The Core Concept

GitLab Flow maps Git branches directly to environments. Developers merge features into the dev branch, which flows into staging after testing, and finally into production once approved. This structured promotion path improves deployment safety.

Workflow Steps

1

Create a Feature Branch

Branch off from dev to build new features or fixes.

2

Merge to Dev

Submit a merge request into the dev branch for initial integration and review.

3

Promote to Staging

After testing in dev, changes are merged into staging for QA and user validation.

4

Promote to Production

Once approved in staging, changes are deployed by merging into production.
