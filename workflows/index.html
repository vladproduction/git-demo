Git Workflows

A collection of visual explanations for the most popular Git workflows used in software development teams.

Centralized Workflow

SVN-style single-source model where all changes go to the main branch.

Integration Manager Workflow

Contributors fork and push changes via pull requests, with a single integrator.

Dictator and Lieutenants

Used in large-scale projects, changes flow from devs to lieutenants to dictator.

Forking Workflow

Ideal for open source; each contributor works in their fork and sends PRs.

GitHub Flow

A simple branch-based workflow perfect for CI/CD and web services.

GitLab Flow

Environment-based workflow with promotion paths like dev → staging → prod.

Trunk-Based Development

All work goes into trunk (main) with short-lived branches and continuous integration.

Feature-Branch Workflow

A simple and effective workflow where all feature development takes place in dedicated branches, keeping the `main` branch stable.

Gitflow Workflow

A structured and robust Git branching model for managing larger projects. Gitflow is ideal for projects with a scheduled release cycle.
