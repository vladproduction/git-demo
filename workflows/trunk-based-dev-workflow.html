Trunk-Based Development

A CI-first strategy where all developers work off a single branch (the "trunk"), merging small, frequent changes with feature toggles and automation to enable rapid integration and deployment.

Visual Workflow Diagram

The Core Concept

Everyone commits directly or via short-lived branches into trunk (usually main). Feature flags and extensive automation allow unfinished work to exist in the trunk without impacting production. This model promotes fast feedback, integration, and deployment.

Workflow Steps

1

Short-Lived Branches

Create quick branches or commit directly to main for fast iteration.

2

CI Pipelines

Every commit triggers builds, tests, and static checks automatically.

3

Feature Flags

Use toggles to control feature rollout without long-lived branches.

4

Deploy Continuously

Production deploys happen frequently, often daily or multiple times per day.
