GitHub Flow

A lightweight, branch-based workflow that supports continuous delivery and is perfect for web apps and services that deploy often.

Visual Workflow Diagram

The Core Concept

GitHub Flow is based on always deploying from main. Developers create short-lived feature branches, push to remote, open pull requests, and deploy merged code. Continuous integration and review are key to the process.

Workflow Steps

1

Create a Branch

Start a new branch from main to work on your feature or fix.

2

Commit Frequently

Push your local commits often to keep your branch updated remotely.

3

Open a Pull Request

Start a pull request for discussion, review, and integration into main.

4

Merge & Deploy

After approval, merge into main and deploy immediately or via CI/CD pipeline.
