Centralized Workflow

A simple, SVN-like approach where a single central repository acts as the single source of truth, and developers push their changes to the `main` branch.

The Core Concept

Unlike branching workflows, the Centralized Workflow doesn't use isolated branches for development. Instead, every team member works directly on the main branch. It mirrors the classic workflow of Subversion (SVN) but with the benefits of Git's local repositories.

Developer Workflow

1

Sync with Central

Start by cloning the central repository or, if you already have it, running git pull to get the latest changes.

2

Make Changes

Edit, add, and delete files to implement the desired feature or fix.

3

Commit Locally

Commit your changes to your local repository. You can make as many commits as you need.

4

Push to Central

Once ready, push your commits to the central repository. If others have pushed changes, Git will require you to pull and resolve conflicts locally before pushing.

Basic Git Commands

Clone Repository: 
git clone https://github.com/your-org/project.git
Get Latest Changes: 
git pull origin main
Stage & Commit Changes: 
git add .
git commit -m "Your message"
Push to Central: 
git push origin main
